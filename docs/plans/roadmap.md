# MVP Roadmap

## Stage 1: Project Setup

- Prepare the GitHub repository.
- Create the basic project structure.
- Add README.md for human users.
- Add AGENTS.md for AI assistants.
- Create the `/docs` folder for project documentation.

## Stage 2: Project Planning

- Define the main goal of the software product.
- Describe the project management approach.
- Split the project into small development stages.
- Decide what features are required for the MVP.
- Decide what features can be moved to future releases.

## Stage 3: MVP Development

The MVP should include only the most important features needed to demonstrate the main idea of the project.

MVP features:

- basic working application structure;
- simple user interface;
- main functionality of the product;
- basic data storage if needed;
- simple error handling;
- basic testing of the main features.

## Stage 4: Testing and Review

- Test the main user flow.
- Check if all MVP features work correctly.
- Fix critical bugs.
- Review the code structure.
- Update documentation if the project changes.

## Stage 5: Final Submission

- Check that all required files are committed to GitHub.
- Make sure the repository is public.
- Check that AGENTS.md, `/docs/pm_approach.md`, and `/docs/plans/roadmap.md` are available.
- Submit the GitHub repository link.

## Future Releases

Features that can be added after the MVP:

- improved user interface;
- more advanced functionality;
- better data validation;
- more detailed statistics or reports;
- additional tests;
- export/import functionality;
- user settings or customization.
