# MVP Roadmap

## Project Goal

The goal of this project is to improve and finalize the **Millionaire Quiz** desktop application. The project should demonstrate a working Python application with a clear structure, graphical user interface, SQLite database usage, and additional features required for the Software Engineering course.

## Stage 1: Project Setup

- Prepare the GitHub repository.
- Keep the existing Millionaire Quiz project structure.
- Add README.md for human users and the instructor.
- Add AGENTS.md for AI assistants.
- Create the `/docs` folder for project management documentation.
- Create the roadmap for MVP and future releases.

## Stage 2: Current Application Review

- Review the existing Python files.
- Check how Tkinter screens are organized.
- Check how SQLite database is used.
- Check the current question system.
- Identify which parts can be improved without rewriting the whole project.

## Stage 3: MVP Development

The MVP should include the main working version of the quiz game with the most important improvements.

MVP features:

- start screen with player name;
- quiz gameplay with multiple questions;
- answer selection with four answer options;
- prize progression;
- option to take money after reaching the required level;
- SQLite database for questions;
- admin panel for adding, editing, and deleting questions;
- search and filtering for questions;
- basic statistics for questions;
- improved interface layout;
- basic error handling.

## Stage 4: Course Improvement Features

These features should be added or improved during the course work:

- game settings:
  - choose number of questions: 5, 10, 15, or 20;
  - choose difficulty mode: easy, medium, hard, or mix;
- lifelines:
  - 50/50 removes two wrong answers;
  - Second Chance allows one mistake before the final answer is accepted;
- game history in SQLite:
  - save player name;
  - save question text;
  - save selected answer;
  - save correct answer;
  - save prize/result;
  - show correct answers visually;
- leaderboard:
  - show best players;
  - sort by highest prize;
  - show date and final result;
- CSV import/export:
  - import questions from CSV;
  - export question bank to CSV.

## Stage 5: Testing and Review

- Test the main quiz flow.
- Test wrong and correct answer scenarios.
- Test the “take money” option.
- Test game settings.
- Test lifelines.
- Test SQLite saving and loading.
- Test leaderboard results.
- Test CSV import/export.
- Fix critical bugs.
- Review code readability and project structure.
- Update documentation if project features change.

## Stage 6: Final Submission

- Check that all required files are committed to GitHub.
- Make sure the repository is public.
- Check that AGENTS.md exists in the root folder.
- Check that `/docs/pm_approach.md` exists.
- Check that `/docs/plans/roadmap.md` exists.
- Make sure README.md explains the project clearly.
- Submit the GitHub repository link.

## Future Releases

Features that can be added after the MVP:

- more modern visual design;
- more question categories;
- sound effects;
- better animations;
- user profiles;
- more detailed statistics;
- export game history;
- online question updates;
- more advanced admin tools.
