# Project Management Approach

This project will use a hybrid project management approach. The work will be divided into small stages similar to Scrum, but the task flow will also stay flexible like Kanban. This approach is suitable because the project can grow step by step, and some features may be improved or changed during development.

The human developer is responsible for the main project idea, architecture decisions, final code review, testing the final product, and checking that the project meets the course requirements. AI assistants can help with boilerplate code, documentation drafts, unit tests, small refactoring tasks, and explaining possible solutions. However, AI should work inside the limits of the project documentation and should always check the `/docs` folder before making changes.
